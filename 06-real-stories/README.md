# Real Stories

This section is where the guide gets honest.

Most beginner guides make agent setup look cleaner than it really is. They show the happy path: install the tool, write a config file, run a command, done.

That is not how our setup worked.

We had agents crash, fill their memory, misunderstand providers, and need help from other agents. None of that means the system was useless. It means agent setup is a real operating practice, not a magic switch.

These stories explain what happened, how we fixed it, and what a beginner should learn from it.

## People and agents in these stories

- Bill Watson - human owner and operator
- Hermes - Ubuntu agent instance
- Alex - OpenClaw Coordinator Agent
- Hannah - Hermes agent instance running on a separate Windows system

## How to read these

Each story follows the same pattern:

1. what happened
2. why it mattered
3. what fixed it
4. what a beginner should do differently

The point is not to embarrass the agent.

The point is to show that mistakes are normal, recoverable, and useful when they turn into better rules.
