# Beginner's Guide to Configuring Agents

**You installed an AI agent. Now what?**

This is a beginner-friendly, hype-free guide to setting up and configuring your first local AI agent harness. We use [Hermes Agent](https://hermes-agent.nousresearch.com) as the worked example throughout — but the concepts, patterns, and lessons transfer to any local agent harness you might use now or in the future.

## Who this is for

- **Complete beginners** who've never installed a local AI agent and want to know what they're getting into
- **Claude Code or Codex users** who are ready to graduate to a full local harness but don't know what that involves
- **Anyone who installed an agent** and is frustrated that it doesn't seem to "get" them or know what to do

If that's you — welcome. You're in the right place.

## The honest pitch

Here's what nobody tells you when you install a local agent:

**It doesn't just work.** Not because the software is bad, but because a fresh install knows nothing about you. It doesn't know your name, your projects, your preferences, your setup, or what you're trying to accomplish. It's a smart person with amnesia — capable, but starting from zero every single session.

The good news: this is fixable. And it's not as hard as it sounds.

This guide walks you through everything that happens *after* the install command finishes — the stuff that makes the difference between an agent that feels useless and one that feels like a real partner.

## Our approach

**Real over polished.** We write about things that actually happened to us — crashes, mistakes, confusion, fixes. Not theoretical best practices.

**Encouraging over intimidating.** You don't need to be a developer to do this. If you can follow a recipe, you can configure an agent.

**Honest about the fear.** Letting AI touch your computer is a real decision. We address the security concerns directly — no hype, no fear-mongering, no pretending the risks don't exist.

**Show, don't just tell.** Every section includes actual prompts, actual commands, and actual outcomes. You can copy-paste and follow along.

## A note on scope

This guide focuses on **configuration and customization** — the stuff that happens after installation. We use Hermes Agent as our example because it's what we use. But the patterns here (context files, memory management, scheduled maintenance, the buddy system) apply broadly to any local agent harness.

If you're using a different harness, the concepts transfer. The specific commands and file paths won't — but the *what* and *why* will.
